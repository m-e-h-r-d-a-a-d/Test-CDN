Manual CDN Verification Guide

← Back to Dashboard

Follow these steps to verify your CDN setup (VergeCloud and Anonymous CDN) end‑to‑end. Keep an SSH session open to tail nginx logs on the origin while running the commands below.

1) Tail nginx logs on origin

Run this to observe incoming requests while you test:

ssh ubuntu@64.227.143.158 'docker logs -f --tail=200 cdn-test-nginx'

2) DNS sanity

Both resolvers should return CDN POP addresses, not your origin IP.

nslookup test-verge-test.shop 1.1.1.1
nslookup test-verge-test.shop 8.8.8.8
nslookup test20250316.ir 1.1.1.1
nslookup test20250316.ir 8.8.8.8

3) Origin reachability with Host override

Confirm origin serves content when addressed by the CDN Host header (HTTP and HTTPS with self‑signed cert).

curl -I -H "Host: test-verge-test.shop" http://64.227.143.158/
curl -skI -H "Host: test-verge-test.shop" https://64.227.143.158/

curl -I -H "Host: test20250316.ir" http://64.227.143.158/
curl -skI -H "Host: test20250316.ir" https://64.227.143.158/

Expect 200, and origin headers like X-Served-By, Cache-Control, CORS headers. Check the nginx logs for matching requests.

4) CDN reachability + SSL

Hit the CDN endpoints directly:

curl -I http://test-verge-test.shop/
curl -I https://test-verge-test.shop/

curl -I http://test20250316.ir/
curl -I https://test20250316.ir/

If HTTPS fails and the origin log shows nothing, use Flexible SSL (origin http:80). If requests reach origin but fail, keep Flexible or use Full (not Strict) unless you install a proper origin cert for the hostname.

5) Response headers

curl -sI https://test-verge-test.shop/ | tr -d '\r'
curl -sI https://test20250316.ir/   | tr -d '\r'

Look for CDN indicators (Via, vendor headers), and origin headers (X-Served-By, X-Origin-Host).

6) Quick timings

curl -o /dev/null -s -w "ttfb: %{time_starttransfer}s total: %{time_total}s\n" https://test-verge-test.shop/
curl -o /dev/null -s -w "ttfb: %{time_starttransfer}s total: %{time_total}s\n" https://64.227.143.158/
